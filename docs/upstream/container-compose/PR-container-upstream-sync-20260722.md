## Motivation and Context

Compose packages a separately versioned Container runtime. Its manifest,
resolved dependency graph, and release provenance must refer to the same
immutable Container revision. This change advances that one revision from the
previous fork tip to the signed merge of Apple Container main through
`f4757afa`.

The change intentionally does not copy or reshape Apple runtime behavior in
Compose. The Apple-facing merge remains isolated in Container; Compose merely
consumes the generic runtime APIs and keeps Docker Compose compatibility policy
at the Compose boundary.

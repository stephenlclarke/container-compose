## Motivation And Context

Optional plugin discovery should not make root help appear hung when the API
service accepts XPC requests but does not answer. The generic timeout must also
return on cancellation without closing a reusable client connection.

## Upstream Delta Decision

The first commit imports `apple/container#1862` unchanged on current Apple
`main`, preserving its original author and implementation. The second commit
adds deterministic coverage for its cancellation behavior. The final commit is
the help-specific policy delta. Once `apple/container#1862` lands, rebase and
drop the imported commit.

## What Changed

- Make XPC timeout and caller cancellation return promptly.
- Reply `invalidArgument` for unknown XPC routes.
- Ignore late replies safely and keep the client reusable.
- Use a one-second API health deadline for root help plugin discovery.
- Preserve the normal ten-second plugin dispatch deadline.
- Fall back to built-in root help when plugin discovery is unavailable.
- Cover root `--help`, `help`, bare invocation, and debug help.

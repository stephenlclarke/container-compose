## Motivation and Context

This is the smallest upstream-shaped health slice needed before `apple/container` can support Docker Compose `depends_on.condition: service_healthy` through direct APIs. It is based on Chris George's draft [apple/container#1504](https://github.com/apple/container/pull/1504) and companion issue [apple/container#1502](https://github.com/apple/container/issues/1502).

External orchestrators need a stable container health field, but the runtime observer design should remain separate. A healthcheck observer needs maintainers to settle where probe configuration lives, who executes probes, how state crosses the sandbox/API-server boundary, and how typed cadence/retry/start-period behavior should be represented.

This PR reserves only the public data shape:

- `HealthStatus`
- `ContainerSnapshot.health`

The field remains `nil` until a later runtime observer PR populates it.

## What Changed

- Adds `Sources/ContainerResource/Container/HealthStatus.swift` with the reserved `none`, `starting`, `healthy`, and `unhealthy` cases.
- Adds optional `health: HealthStatus?` to `ContainerSnapshot`.
- Defaults `health` to `nil` in the `ContainerSnapshot` initializer.
- Extends `ContainerSnapshotTests` so legacy snapshots decode with `health == nil`.
- Adds a Codable round-trip test for every reserved health status.

## Compatibility Notes

`ContainerSnapshot` is marshaled as `Codable` JSON. Adding an optional field is wire-compatible:

- Older clients reading from a newer daemon can ignore the new key.
- Newer clients reading from an older daemon decode `health` as `nil`.

The next PR-shaped slice should add the actual runtime observer and populate this field. Released `container-compose` support for `service_healthy` remains blocked until both the data shape and observer semantics are available upstream or in the fork.

## Remaining Risks

- Maintainers may prefer to defer the public field until the observer design lands.
- The observer follow-up still needs typed decisions for `interval`, `timeout`, `retries`, `start_period`, exit-code handling, and when `.starting` transitions to `.unhealthy`. Docker/Compose field parsing belongs in `container-compose`.
